# Principles

This reference collects the strongest reusable ideas for designing mobile interfaces that feel distinctive, trustworthy, and cohesive.

## Core Mental Model

High-value mobile UI is not just polished.

It is recognizable, easy to use, and consistent enough that the product feels trustworthy.

Three foundational pillars:

1. **Identity**
- A screenshot should feel like this product, not a template.
- Identity can come from typography, accent strategy, icon rhythm, illustration style, composition, or a memorable interaction signature.
- Distinctive does not mean loud.

2. **Familiarity**
- Users should recognize the interaction model quickly.
- Copy patterns from products the audience already knows when that lowers learning cost.
- Avoid innovation in foundational interaction patterns unless the gain is obvious.

3. **Consistency**
- The same product should not look like five different designers worked on five different screens.
- Spacing, type scale, color roles, corner radii, icon style, and motion rhythm must repeat predictably.

## Distilled Design Heuristics

- Use thumb-zone awareness and strong screen hierarchy to keep mobile interaction low-friction
- Separate build mode from review mode so creation and auditing do not blur together
- Prioritize accessibility, touch, layout, and system consistency before decorative polish
- Avoid generic AI aesthetics by committing to a deliberate visual direction
- Use implementation-aware token systems so the design survives handoff into code

## Premium Value Rules

- Premium perception usually comes from restraint, not more effects
- Remove design noise before adding polish
- Use one focal point per screen
- Use typography and spacing to create confidence
- Use motion as reassurance, not spectacle

## AI-Native Product Rules

- Chat is familiar, so it reduces onboarding friction
- Generative UI is powerful when it turns text intent into actionable structure
- A component should appear only when it helps the user do something faster than prose
- Inline UI must preserve control, context, and fallback behavior

If the work starts with gradients, cards, and animation choices, it started too late.

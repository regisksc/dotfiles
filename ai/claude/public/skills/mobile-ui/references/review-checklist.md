# Review Checklist

Use this when auditing existing mobile UI.

## Critical

- [ ] Primary action is obvious within two seconds
- [ ] Tap targets are comfortable and readable
- [ ] Contrast is sufficient and body text is readable
- [ ] Loading, empty, error, success, and disabled states exist
- [ ] Navigation follows familiar mobile conventions
- [ ] No interaction requires unnecessary learning effort

## High Priority

- [ ] Screens share the same spacing and radius system
- [ ] Colors have semantic roles, not random screen-by-screen usage
- [ ] Typography scale is consistent
- [ ] Icons come from one family
- [ ] Motion is restrained and purposeful
- [ ] One memorable cue exists without overwhelming the screen

## AI-Native Checks

- [ ] Chat is used where familiarity lowers friction
- [ ] Generative UI appears only when it reduces effort
- [ ] Inline components are controllable, legible, and reversible
- [ ] Loading skeletons or placeholders support streamed UI
- [ ] User feedback can be captured close to the result

## Premium Value Checks

- [ ] The product does not resemble a default AI template
- [ ] The screen feels intentional, not assembled from random cards
- [ ] The visual hierarchy is calm and credible
- [ ] The design language feels like one product across multiple screens

## Findings Format

Report:

1. Finding
2. Why it weakens value or usability
3. Priority
4. Fix direction
